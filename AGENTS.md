# Agent Handoff

このリポジトリをレビュー・修正するエージェントは、最初にこのファイルを読んでください。

## 目的

公明新聞の広告PDFを日別に取り込み、同日内で広告主・業種・訴求内容がかぶって掲載上問題になりそうな広告を見つけるための簡素な試作サイトです。

## 重要な前提

- 対象PDFはスキャンPDFです。埋め込みテキストは抽出できません。
- 初版はブラウザだけで動く静的サイトです。
- 本番メール送信、サーバー保存、サーバーOCR、外部AI API連携は未実装です。
- 判定は初期案であり、最終判断は担当者フィードバックで調整します。
- private chain-of-thought は共有しません。代わりに判断理由、実装意図、変更履歴を文書化しています。

## レビュー観点

- PDF取り込み後、ページが縦向きに補正されるか。
- 1面の題字横、題字中、記事下メインを扱えるか。
- ラテ面とラテ中広告を扱えるか。
- 8面固定になっておらず、実PDFのページ数に追従できるか。
- フィードバック登録後に入力欄が消えるか。
- 過去フィードバックが履歴として読めるか。
- メールアドレス登録とアラート文面作成が自然か。
- `○△✕` 判定理由が担当者に説明できるか。
- 静的試作の範囲と本番化で必要な範囲が混ざっていないか。
